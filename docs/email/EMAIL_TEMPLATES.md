# Email-шаблоны: Granite CRM

> Каталог всех email-шаблонов для рассылок и post-reply ответов.
> **Источник истины:** этот файл. Тексты правит Александр прямо здесь.
> Рабочие шаблоны загружаются из `data/email_templates.json` (TemplateRegistry).
> Для перезагрузки без рестарта: `POST /api/v1/templates/reload` или `uv run cli.py templates-reload`
>
> Плейсхолдеры: `{city}`, `{city_locative}`, `{from_name}`, `{whatsapp_number}`, `{telegram_link}`, `{unsubscribe_url}`, `{original_subject}`
>
> `{city_locative}` — город в предложном падеже (Москве, Казани), подставляется автоматически
> из словаря `data/city_declensions.json` через `granite/city_declensions.py`.
> После предлога «в» **всегда** используйте `{city_locative}`, а не `{city}`.
>
> **Принципы:** все шаблоны — `body_type: plain`. Письмо должно выглядеть как личная переписка,
> а не как HTML-рассылка. Без маркированных списков с жирным текстом (`**текст**`),
> без «мы/наши» (исполнитель — один человек, самозанятый), без эмоциональных крючков.
>
> **Структура холодного письма:** Ценность → СК → Детали → Условия → Контакты.
> СК (бесплатный тест на исходнике) — в первом экране, не ниже второго абзаца.
>
> **{company_name} НЕ используется** в холодных письмах — автоматически никогда,
> вручную только после проверки сайта (парсер часто выдаёт SEO-мусор,
> а «Здравствуйте Мастер-гранит» выглядит как 100% спам).
>
> **Отписка:** единый механизм — «Не актуально — отписаться: {unsubscribe_url}».
> Никаких «ответьте нет» рядом со ссылкой — два действия путают клиента.
>
> Подробнее: `guide-email-templates.md` в download/.

## 1. Холодные письма (Cold emails)

```yaml
body_type: plain

```

```
для мастерских в {city_locative}.

покажу результат.

до чёткого портрета под ваш станок (лазер/ударник).
Срок — до 12 часов, срочно — 3–6 часов.
Цена — от 700 ₽, оплата после одобрения результата.
Работаю официально, выдаю чеки (НПД).

Примеры работ: https://retouchgrav.netlify.app
```

## 5. Волновые темы для A/B тестирования

При создании кампании можно задать `subject_a` и `subject_b` для A/B теста. Тело письма одно — `template_name`.

### Волна 1: Основной оффер

| Вариант | Тема | Гипотеза |
|---------|------|----------|
| A | «Подготовка фото под гравировку» | Предметная, нейтральная — выглядит как деловое письмо |
| B | «Ретушь портретов для вашей мастерской» | Акцент на услугу + персонализация |

### Волна 2: Сложные случаи

| Вариант | Тема | Гипотеза |
|---------|------|----------|
| A | «Сложные фото для гравировки — бесплатный тест» | Акцент на сложные случаи + бесплатный тест |
| B | «Бесплатный тест: покажу результат за 2 часа» | Акцент на бесплатный тест + скорость |

### Волна 3: Сезонная нагрузка

| Вариант | Тема | Гипотеза |
|---------|------|----------|
| A | «Ретушь для мастерских в {city_locative} — оплата по результату» | Локальная привязка + снятие риска |
| B | «Ретушь фото в день обращения — бесплатный тест» | Акцент на скорость + бесплатный тест |

## 7. Справочник плейсхолдеров

| Плейсхолдер | Описание | Пример подстановки |
|---|---|---|
| `{from_name}` | Имя отправителя | Александр |
| `{whatsapp_number}` | WhatsApp-номер отправителя (из config.yaml → sender.whatsapp) | +7 900 143-00-78 |
| `{telegram_link}` | Telegram-ссылка отправителя (из config.yaml → sender.telegram) | https://t.me/ganjavagen |
| `{city}` | Название города (именительный падеж) | Екатеринбург |
| `{city_locative}` | Город в предложном падеже (после «в») | Екатеринбурге |
| `{original_subject}` | Тема предыдущего письма контакту (из `CrmTouchRow`, для Re:) | Вопрос по ретуши |
| `{unsubscribe_url}` | Ссылка на отписку | https://… |

**Важно:** `{city_locative}` вычисляется автоматически через `get_locative(city)` из модуля `granite/city_declensions.py` с данными из `data/city_declensions.json` (1093 города). После предлога «в» всегда используйте `{city_locative}`, а не `{city}`. Если город отсутствует в словаре — fallback на именительный падеж.

**⚠️ {company_name} НЕ используется в шаблонах.** Парсер часто определяет название компании неверно (SEO-мусор вместо реального имени). Автоматическая подстановка «Здравствуйте {company_name}» выглядит как спам. Использовать только вручную после проверки сайта.

## 8. Чеклист перед отправкой

- [ ] Шаблоны загружены (GET /api/v1/templates показывает полный список)
- [ ] Плейсхолдеры корректны (нет `{company_name}` без данных)
- [ ] `{city_locative}` resolves correctly — город есть в `city_declensions.json`, иначе fallback на именительный
- [ ] После «в» используется `{city_locative}`, не `{city}`
- [ ] Нет `**жирного текста**` в plain-шаблонах (отобразится как мусор)
- [ ] Нет «мы/наших» — только «я» (исполнитель — самозанятый)
- [ ] Нет «Ищу» — только «Делаю» (позиция предложителя, не просителя)
- [ ] СК (бесплатный тест) — в первом экране, не ниже 2-го абзаца
- [ ] Нет «нейросети» / «AI» — только результат («восстановлю до чёткого портрета»)
- [ ] НПД есть во всех холодных письмах
- [ ] WA-ссылка есть в подписи каждого email-шаблона
- [ ] Подпись единообразна: `{from_name}` + TG + WA
- [ ] Отписка единая: «Не актуально — отписаться: {unsubscribe_url}»
- [ ] Длина в пределах нормы: cold 60–80 слов, follow-up 40–60, post-reply 30–50
- [ ] A/B темы заданы для кампании
- [ ] Тестовое письмо отправлено на свой email
- [ ] Unsubscribe URL работает
- [ ] После редактирования JSON — перезагрузка: POST /api/v1/templates/reload
