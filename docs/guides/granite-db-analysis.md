## 2. Схема данных и связи

```
raw_companies ──(дедуп)──► companies ◄──── enriched_companies
                                 │
                                 ├── crm_contacts (1:1)
                                 ├── crm_tasks (1:N)
                                 ├── crm_touches (1:N)
                                 └── crm_email_logs (1:N)

cities_ref ── справочник для городов компаний

crm_email_campaigns ──(1:N)──► crm_email_logs

crm_templates ── независимый справочник шаблонов

unmatched_cities ── лог нераспознанных городов
```

## 8. Ключевые выводы и рекомендации

### Что работает хорошо
1. **Парсинг собрал обширную базу** — 11 854 сырых записей из двух источников.
2. **Дедупликация отработала** — удалено ~27.6% дублей, итог 8 587 компаний.
3. **Обогащение проведено почти для всех** — 8 585 из 8 587 компаний имеют enriched-данные.
4. **Скоринг и сегментация выполнены** — 82% компаний получили сегмент.
5. **Справочник городов заполнен** — 1 093 города, все успешно обработаны.

### Проблемы и узкие места
1. **Email-покрытие критически низкое** — лишь 28.5% компаний имеют email. Для email-кампаний этого мало. Рекомендуется: парсинг email с сайтов, дозаполнение через enrichment.
2. **2 625 компаний нуждаются в ручной проверке** (`needs_review = 1`) — 30.6% базы. Требуется пайплайн обработки.
3. **1 533 компании без сегмента** — необходимо дообогащение или ручная классификация.
4. **CRM-воронка не запущена** — все контакты на стадии `new`, ни одного касания, задачи, шаблона.
5. **Слияние дублей не производилось** — `merged_into = 0` везде. Возможны необнаруженные дубли внутри `companies`.
6. **CMS не определена у 75.6%** — можно улучшить детекцию или пометить как «нет сайта / конструктор».

### Приоритетные действия
1. **Дособрать email-адреса** — главный блокер для запуска email-кампаний.
2. **Обработать очередь needs_review** — почти треть базы ждёт ручной проверки.
3. **Запустить воронку** — перевести сегмент A (1 570 компаний) в первый этап обработки.
4. **Создать шаблоны** (`crm_templates`) для email и мессенджеров.
5. **Улучшить детекцию CMS** — 75% unknown — слишком много для аналитики.
