## Дизайн-система Granite CRM: Современный стандарт для B2B-интерфейсов

Дизайн-система Granite CRM разработана для обеспечения высокой производительности, ясности данных и эффективности рабочих процессов. Она ориентирована на профессиональное использование, где скорость обработки информации и точность действий имеют первостепенное значение. Система построена на принципах минимализма, высокой контрастности и строгой иерархии.

### 1. Основы и принципы

В основе дизайн-системы лежат три ключевых принципа: **Эффективность**, **Прозрачность** и **Надёжность**. Интерфейс не должен отвлекать пользователя; он должен служить инструментом для быстрого принятия решений.

| `--radius-sm` | 4px | Маленькие badge, tag |
| `--radius-md` | 6px | Кнопки, инпуты, фильтры |
| `--radius-lg` | 8px | Карточки, модалки |
| `--radius-xl` | 10px | Резерв |

#### 1.3 Типографика

Используется современный гротеск **Inter** (или системный шрифт), который обеспечивает отличную читаемость в таблицах и интерфейсных элементах.

*   **Заголовки:** Semibold (600), `--foreground`.
*   **Основной текст:** Regular (400), `--foreground`, 14px (стандарт для CRM).
*   **Мелкий текст/Подписи:** Medium (500), `--muted-foreground`, 12px.
*   **Моноширинный (для ID/Кода):** JetBrains Mono, 13px.

### 2. Компоненты интерфейса

Компоненты спроектированы с учётом библиотеки **shadcn/ui** и адаптированы под специфику CRM.

#### 2.1 Кнопки и Инпуты

Кнопки имеют скругление `rounded-md` (6px, `--radius-md`) и чёткие состояния (hover, active, disabled). Инпуты используют тот же радиус. Для карточек используется `rounded-xl` (8px, `--radius-lg`).

#### 2.2 Badge

Badge — компактный индикатор статуса, метки или значения. Использует компонент `Badge` из `@/components/ui/badge`.

**Размеры (prop `size`):**

| Размер | h | px | font | Применение |
| :--- | :--- | :--- | :--- | :--- |
| `sm` | `h-4` | `px-1.5` | `text-[10px]` | Статусы в таблицах, ID, фильтры |
| `default` | `h-5` | `px-2` | `text-xs` | Метки по умолчанию, счётчики |
| `lg` | `h-6` | `px-3` | `text-sm` | Сегменты в карточках/Sheet, хедеры |

**Варианты (prop `variant`):**

| Вариант | Фон | Текст | Граница | Применение |
| :--- | :--- | :--- | :--- | :--- |
| `default` | `--primary` | `--primary-foreground` | нет | Статус компании, каналы, активное |
| `secondary` | `--secondary` | `--secondary-foreground` | нет | Второстепенные метки |
| `destructive` | `--destructive/10` | `--destructive` | нет | Ошибки, баунс, неуспешные отправки |
| `outline` | прозрачный | `--foreground` | `--border` | **Используется чаще всего**: ID, счётчики, теги, метрики, "Не отправлено" |
| `success` | `--success/10` | `--success` | `--success/20` | Отправлено, успех |
| `ghost` | прозрачный (hover `--muted`) | `--foreground` | нет | Ховер-метки |
| `link` | нет | `--primary` | нет | Ссылочные индикаторы |

**Семантика для статусов рассылки:**

| Статус | Badge |
| :--- | :--- |
| В очереди | `<Badge variant="secondary" size="sm">` |
| Отправлено | `<Badge variant="success" size="sm">` |
| Открыто | `<Badge variant="outline" size="sm" className="bg-blue-100/50 dark:bg-blue-900/20 text-blue-700 dark:text-blue-400">` |
| Ответили | `<Badge variant="outline" size="sm" className="bg-purple-100/50 dark:bg-purple-900/20 text-purple-700 dark:text-purple-400">` |
| Не доставлено | `<Badge variant="destructive" size="sm">` |
| Ошибка | `<Badge variant="destructive" size="sm">` |
| Не отправлено | `<Badge variant="outline" size="sm">` |

**Правила:**
1. Не использовать `text-[*]`, `h-*`, `px-*`, `py-*` — размеры задаются через `size`.
2. Для моноширинного текста (Score, ID) добавить `className="font-mono"`.
3. `font-mono-code` устарел — использовать `font-mono`.
4. Badge с кастомными цветами (blue, purple, amber) — всегда `variant="outline"` + цветовые классы.

#### 2.3 Таблицы (Data Tables)

Таблица — сердце CRM. Она должна быть максимально информативной, но не перегруженной.

*   **Зебра-раскраска:** Отсутствует, используется разделение тонкими линиями.
*   **Hover-эффект:** Подсветка всей строки при наведении.
*   **Плотность:** Compact (высота строки 40-48px).
*   **Индикаторы:** Использование Badge для стадий воронки (например, `new` — Gray, `interested` — Green).

### 3. Паттерны макета и навигации

#### 3.1 Sidebar (Боковая навигация)

Узкий Sidebar (240px) с тёмным или очень светлым фоном. Иконки — контурные (Lucide React), размер 20px. Активный пункт выделяется заливкой или левой акцентной линией.

#### 3.2 Side Panel (Детальная информация)

Вместо перехода на новую страницу для просмотра деталей компании используется **Side Panel (Sheet)**. Она выезжает справа, перекрывая часть таблицы, что позволяет сохранять контекст работы.

### 4. Визуализация данных и обратная связь

#### 4.1 Воронка продаж (Funnel Chart)

Визуализируется в виде горизонтальных блоков с убывающей шириной. Каждая стадия имеет свой оттенок, плавно переходящий от холодного (новые) к тёплому или яркому (заинтересованные).

#### 4.2 Тени

Тени централизованы в CSS-переменных для единообразия между светлой и тёмной темами:

| Токен | Light | Dark |
| :--- | :--- | :--- |
| `--shadow-sm` | `0 1px 2px rgba(44,41,38,0.06)` | `0 1px 2px rgba(0,0,0,0.3)` |
| `--shadow-md` | `0 4px 6px -1px rgba(44,41,38,0.08)` | `0 4px 6px -1px rgba(0,0,0,0.35)` |
| `--shadow-lg` | `0 10px 15px -3px rgba(44,41,38,0.08)` | `0 10px 15px -3px rgba(0,0,0,0.35)` |
| `--shadow-xl` | `0 20px 25px -5px rgba(44,41,38,0.1)` | `0 20px 25px -5px rgba(0,0,0,0.4)` |

В коде используются Tailwind-классы `shadow-sm`, `shadow-md` и т.д.

#### 4.3 Статусы и уведомления (Toasts)

Для подтверждения действий (например, "Email отправлен") используются всплывающие уведомления в правом нижнем углу. Они должны быть лаконичными и исчезать через 3-5 секунд.

### 5. Рекомендации по реализации

При реализации системы на **Tailwind CSS 4** рекомендуется использовать CSS-переменные для тем (Light/Dark) и строго придерживаться сетки 4px (spacing scale).

> "Дизайн CRM — это не про красоту, а про отсутствие трения между пользователем и его данными."

Данная дизайн-система обеспечивает консистентность интерфейса Granite CRM и закладывает фундамент для масштабирования продукта, сохраняя при этом лёгкость и профессиональный вид.
