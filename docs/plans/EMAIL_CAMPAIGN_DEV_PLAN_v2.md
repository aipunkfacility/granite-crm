## Реальная картина базы

Прежде чем планировать — важно понимать ограничения данных.

**Что есть в базе надёжно:** название, город, сайт, телефоны, email, мессенджеры, CMS, crm_score, сегмент, is_network, has_marquiz, tg_trust.

**Чего нет:** информация о наличии гравировального станка, типе оборудования, объёме заказов. Tech_keywords из конфига в базу не попали — сайты мастерских редко пишут об оборудовании в явном виде.

**Главный фильтр — is_network.** Из 578 компаний сегмента A с email — 403 помечены как сети. Это агрегаторы (`danila-master.ru`, `memorial.ru`, `vsepamyatniki.ru`), представленные в десятках городов. Писать им по каждому городу бессмысленно. После фильтрации:

```
Сегмент A, не-сеть, валидный email:  175 компаний
Сегмент B, не-сеть, валидный email:  259 компаний
─────────────────────────────────────────────────
Итого приоритетная база:              434 компании
```

При 50 письмах в день — **8–9 рабочих дней** на первый полный прогон.

**Крупные сети — отдельный разговор.** `pamyatniki.moscow` (9 городов), `ritualgranit.ru` (7 городов), `ramgranit.ru` (7 городов) — им стоит написать **один раз** напрямую в головной офис, вручную. Это потенциальный оптовый клиент с десятками заказов в неделю.

## Сегментация волн рассылки

Рассылаем не всё сразу, а последовательными волнами — от самых горячих к тёплым. Каждая волна — отдельная кампания со своей темой письма.

### Волна 1 — Сигнальные (22 компании)

**Фильтр:** `(has_marquiz=1 OR tg_trust>=2) AND is_network=0 AND segment IN (A,B) AND email`

Мастерские, которые уже думают о маркетинге: поставили квиз на сайт (has_marquiz) или ведут живой Telegram-канал с аватаром и описанием (tg_trust≥2). Они понимают ценность аутсорса — объяснять меньше.

**Тема A:** `Ретушь для гравировки — в день обращения, оплата после`  
**Тема B:** `Для вашей мастерской: подготовка фото под гравировку без предоплаты`

### Волна 2 — Bitrix-мастерские (41 компания)

**Фильтр:** `cms=bitrix AND is_network=0 AND segment IN (A,B) AND email`

Bitrix — признак профессионального бизнеса, часто с менеджером или маркетологом. Они привыкли работать с подрядчиками и принимают решения быстрее. Самый конвертируемый сегмент после волны 1.

Примеры из базы: Гранит-Мастер (Белгород, score 63), Небеса (Брянск, score 57), Кинель-Гранит (Кинель, score 57).

**Тема A:** `{company_name}: готовим фото под гравировку — в день заказа`  
**Тема B:** `Аутсорс ретуши для гранитных мастерских — без предоплаты`

### Волна 3 — Tilda-сайты (25 компаний)

**Фильтр:** `cms=tilda AND is_network=0 AND segment IN (A,B) AND email`

Tilda — современный конструктор, владелец сам делал сайт и думает о подаче. Маркетингово восприимчив. Средний сегмент по прогреву.

**Тема A:** `Фото на памятник: ретушь под ваш станок — смотрите примеры`  
**Тема B:** `Свободная минута? Сделаю пробную ретушь для вашей мастерской`

### Волна 4 — Остальной сегмент A (121 компания)

**Фильтр:** `segment=A AND is_network=0 AND cms NOT IN (bitrix,tilda) AND email`

WordPress, Joomla, OpenCart, unknown CMS. Хороший скор, реальный бизнес — просто меньше сигналов о маркетинговой зрелости.

**Тема A:** `Ретушь портретов для гравировки — бесплатная проба`  
**Тема B:** `Первое фото — бесплатно. Оцените качество без обязательств`

### Волна 5 — Сегмент B (259 компаний)

**Фильтр:** `segment=B AND is_network=0 AND email`

Чуть ниже скор, но 259 компаний — существенный объём. Запускать после анализа результатов волн 1–4, когда будет понятна лучшая тема.

**Тема:** победитель из предыдущих A/B тестов

### Параметры безопасной рассылки

```python
EMAIL_DELAY_MIN       = 45    # сек между письмами (минимум)
EMAIL_DELAY_MAX       = 120   # сек между письмами (максимум)
EMAIL_BATCH_PER_DAY   = 50    # старт; поднять до 80-100 после первой недели
EMAIL_SESSION_GAP_HRS = 2     # минимум между сессиями
MAX_PER_DOMAIN        = 2     # писем на один домен-получатель за сутки
```

Случайная задержка (не фиксированная) — критично. Паттерн "каждые 60 секунд" распознаётся провайдером.

### SMTP и трекинг

**Gmail App Password** (не основной пароль):
`Google Account → Security → 2-Step Verification → App Passwords → Mail`

**Yandex** (если переходить позже):
`Настройки → Безопасность → Пароли приложений`

**Cloudflare Tunnel** — для трекинг-пикселя:
```bash
cloudflared tunnel create granite-crm
cloudflared tunnel route dns granite-crm track.yourdomain.com
cloudflared tunnel run granite-crm   # добавить в автозапуск
```

Важно: трекинг-пиксель блокируется Gmail и Яндекс-почтой в ~40–60% случаев. Это норма — реальное число открытий выше, чем показывает пиксель. Используем как относительный показатель для A/B, не как абсолютный.

### Валидатор перед отправкой

```python
def validate_batch(recipients: list) -> tuple[list, list[str]]:
    """Возвращает (чистый список, список предупреждений)."""
    warnings = []
    clean = []

    for r in recipients:
        # Исключить мусорные названия (слишком длинные, без пробелов)
        if len(r['name']) > 80 or ' ' not in r['name'].strip():
            warnings.append(f"Подозрительное название: {r['name'][:50]}")
            continue

        # Исключить .by домены (Беларусь — другая ниша)
        if r['email'].endswith('.by'):
            warnings.append(f"Зарубежный email пропущен: {r['email']}")
            continue

        # Исключить уже получавших письмо в этой кампании
        if r.get('already_sent'):
            continue

        clean.append(r)

    return clean, warnings
```

### CampaignWizard — шаги

**Шаг 1: Фильтры получателей**
- Сегмент (чекбоксы A / B / C)
- Только не-сети (включено по умолчанию)
- CMS (выпадающий список: все / bitrix / wordpress / tilda)
- Только с Marquiz (переключатель)
- Минимальный score (слайдер 0–70)
- Live-превью: «Найдено X компаний, Y с валидным email»

**Шаг 2: A/B тест тем**
- Тема письма A (обязательно)
- Тема письма B (опционально, для A/B)
- Если B заполнена — показать «50% получат A, 50% получат B»
- Предупреждение при спам-словах в теме

**Шаг 3: Тело письма**
- Textarea для HTML
- Кнопки вставки плейсхолдеров: `{company_name}` `{city}` `{unsubscribe_url}`
- Live-превью справа с реальными данными первого получателя

**Шаг 4: Настройки и запуск**
- Лимит за сессию (50 / 80 / 100)
- Задержка между письмами (45–120 сек, слайдер)
- Список предупреждений от валидатора
- Кнопка «Запустить рассылку»

## Автоматика воронки

После рассылки CRM должна сама двигать компанию по воронке:

```
[отправка]    → funnel_stage: new → email_sent
[открытие]    → funnel_stage: email_sent → email_opened
[3 дня тишины после opened] → создать crm_task:
    type: follow_up
    description: «Написал на email, открыли — попробовать Telegram»
    due_date: now + 3 days
[7 дней тишины после sent, не открыли] → создать crm_task:
    type: follow_up
    description: «Email не открыли — попробовать другую тему или Telegram»
    due_date: now + 7 days
```

Задачи появятся в разделе «Задачи» → удобно видеть очередь follow-up в одном месте.

```
         Тема A vs B — фиксируем открытия
```

## Целевые показатели

| Метрика | Реалистично | Хорошо | Отлично |
|---|---|---|---|
| Open rate (пиксель) | 15–20% | 25–30% | 35%+ |
| Реальные открытия (~2× пикселя) | 30–40% | 50–60% | 70%+ |
| Reply rate | 1–2% | 3–5% | 7%+ |
| Писем в день | 50 | 80 | 100 |
| Ответов за первые 2 недели | 2–4 | 5–10 | 15+ |

**Расчёт по базе (434 компании × reply rate 3%):** ~13 потенциальных клиентов из первого прогона. При среднем чеке 700–1000₽ за ретушь и 5+ заказах в неделю от каждого — это хорошая стартовая точка для постоянных партнёров.

## Риски

| Риск | Mitigation |
|---|---|
| Gmail блокирует аккаунт | Случайные задержки 45–120 сек, не более 50/день в старте, не слать всё за 2 часа |
| Письма в спам | mail-tester.com перед стартом, нет спам-слов, подпись с ИНН |
| Трекинг не работает | Норма — 40–60% клиентов блокируют пиксели. Используем как относительный показатель для A/B, не абсолютный |
| Мусорные названия в теле | Валидатор исключает названия без пробелов и длиннее 80 символов |
| .by адреса (Беларусь) | Автофильтр в валидаторе |
| Компьютер офлайн | Scheduler сохраняет позицию, возобновляет со следующего получателя |
| Крупные сети — дубли | is_network=0 в основных волнах; сети — отдельно, вручную |
