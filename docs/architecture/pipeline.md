# Пайплайн RetouchGrav

> Как работает конвейер обработки данных: от скрапинга до экспорта.

---

## Обзор

Пайплайн — автоматический процесс сбора и обработки данных о гранитных мастерских. Запускается командой `uv run cli.py run "Город"` и проходит 6 фаз, сохраняя прогресс в чекпоинтах.

```
run "Город"
  → Фаза 1: Скрапинг (источники → raw_companies)
  → Фаза 2: Дедупликация (Union-Find → companies)
  → Фаза 3: Обогащение (TG, CMS, мессенджеры → enriched_companies)
  → Reverse Lookup (опционально: 2GIS/Yell для компаний с малыми данными)
  → Детекция сетей (агрегаторы и филиалы)
  → Фаза 5: Скоринг (A/B/C/D сегментация)
  → Фаза 6: Экспорт (CSV + пресеты)
```

---

## Фаза 1: Скрапинг

**Цель:** Собрать сырые данные о компаниях из всех включённых источников.

**Источники (настраиваются в `config.yaml` → `sources`):**

| Источник | Метод | Состояние |
|----------|-------|-----------|
| jsprav.ru | JSON-LD парсинг (быстрый) + Playwright fallback | Включён |
| web_search | DuckDuckGo поиск → скрапинг сайтов | Включён |
| 2GIS | Crawlee + 2GIS Catalog API | Выключен (по умолчанию) |
| Yell.ru | Crawlee PlaywrightCrawler | Выключен (по умолчанию) |

**Результат:** Записи в таблице `raw_companies` (сырые, с дублями).

**Автоматический поиск категорий:** Для jsprav.ru поддомены и категории определяются через API `/api/cities/`, кэшируются в `data/category_cache.yaml`. Ручные замены — через `subdomain_map` в config.yaml.

**Для каждого города в области:** Скраперы обрабатывают все населённые пункты области (из `data/regions.yaml`). Между городами — пауза `inter_city_delay` (2 секунды).

---

## Фаза 2: Дедупликация

**Цель:** Слить дубликаты — одну и ту же мастерскую из разных источников.

**Алгоритм:** Union-Find по общим телефонам и доменам сайта.

1. `phone_cluster.py` — компании с общим телефоном → один кластер
2. `site_matcher.py` — компании с одним доменом → один кластер
3. `merger.py` — слияние записей в кластере: лучшее имя, объединение телефонов/email/мессенджеров
4. `validator.py` — валидация телефонов, email, сайтов

**Настройки (config.yaml → `dedup`):**
- `name_similarity_threshold: 88` — порог fuzzy matching
- `phone_match_required: true` — сливать только при общем телефоне
- `network_phone_threshold: 10` — телефоны у 10+ компаний исключаются из кластеризации (агрегаторы)

**Результат:** Записи в таблице `companies` (уникальные).

---

## Фаза 3: Обогащение

**Цель:** Найти дополнительные данные для каждой уникальной компании.

Два прохода:

### Проход 1: Сканирование сайта и Telegram

Для каждой компании (sync ThreadPoolExecutor или async httpx в зависимости от `config.yaml → enrichment.async_enabled`):

1. **MessengerScanner** — парсит HTML сайта, ищет ссылки t.me, wa.me, vk.com
2. **TgFinder** — ищет Telegram по телефону (t.me/+7XXX) и названию (генерация юзернеймов)
3. **TgTrust** — анализирует профиль TG: аватар (+1), описание (+1), канал (-1), бот (-1). `trust_score ≥ 2` = живой контакт
4. **TechExtractor** — определяет CMS (Bitrix, WordPress, Tilda), ключевые слова (оборудование, производство, ретушь)

### Проход 2: Точечный поиск (web_search)

Для компаний без сайта или email:
- web_search «Название Город» → лучший URL
- Скрапинг URL → извлечение email, телефонов
- Сканирование найденного сайта на мессенджеры и CMS

**Настройки:** `max_concurrent: 3` (потоки), `batch_flush: 50` (сохранять каждые 50 записей)

**Результат:** Записи в таблице `enriched_companies`.

---

## Reverse Lookup (опционально)

**Когда включён:** `config.yaml → enrichment.reverse_lookup.enabled: true`

Для компаний без мессенджеров, без email и с низким crm_score:
- Поиск в Yell через Crawlee PlaywrightCrawler
- Поиск в 2GIS через API или Crawlee (если доступен ключ)
- Слияние найденных данных (без перезаписи существующих)

**Ограничения:** Лимит запросов в день (настраивается в конфиге), задержка между запросами.

## Детекция сетей

**После обогащения, перед скорингом.**

Ищет компании с филиалами (один домен/телефон у 2+ компаний в одном городе). Помечает `is_network = True`, что даёт +5 баллов к скору.

При пакетном запуске (`run all`) после всех городов автоматически запускается `scan-networks` — поиск глобальных сетей (домен в 3+ городах). Такие записи помечаются `needs_review = True`.

## Фаза 5: Скоринг

**Цель:** Числовая оценка и сегментация каждой компании.

**Веса (config.yaml → scoring.weights):**

| Параметр | Баллы |
|----------|-------|
| has_telegram | +15 |
| has_whatsapp | +10 |
| cms_bitrix | +10 |
| has_marquiz | +8 |
| has_website | +5 |
| has_email | +5 |
| multiple_phones | +5 |
| is_network | +5 |
| has_address | +5 |
| cms_modern | +3 |
| tg_trust_multiplier | ×3 (tier-based: 0→-10, 1→3, 2→6, 3→9) |
| site_constructor | -3 |
| TLD-штрафы (.kz, .by, .com, .net) | -3 до -5 |

**Пороги (config.yaml → scoring.levels):**

| Сегмент | Порог | Доля компаний |
|---------|-------|---------------|
| A | ≥ 40 | ~8% |
| B | ≥ 25 | ~20% |
| C | ≥ 15 | ~35% |
| D | < 15 | ~37% |

## Фаза 6: Экспорт

**Автоматический:** CSV при завершении пайплайна → `data/export/{город}_enriched.csv`

**Пресеты (config.yaml → export_presets):**
- `hot_leads` — TG/WA + высокий скор
- `high_score` — сегмент A
- `with_telegram` — все с Telegram
- `cold_email` — нет мессенджеров, есть email
- `manual_search` — нет мессенджеров (Markdown)
- `full_dump` — все компании

## Чекпоинты

Пайплайн запоминает прогресс. При перезапуске — продолжает с прерванного этапа.

| Этап | Что проверяется | Следующая фаза |
|------|-----------------|----------------|
| `start` | raw_companies = 0 | Скрапинг |
| `scraped` | raw_companies > 0 | Дедупликация |
| `deduped` | companies > 0 | Обогащение |
| `enriched` | enriched_companies > 0 | Скоринг + экспорт |

**Флаги:**
- `--force` — очистить все данные по городу, начать с нуля
- `--no-scrape` — пропустить скрапинг, начать с дедупликации
- `--re-enrich` — пропустить скрапинг и дедупликацию, запустить только обогащение

## Типичные проблемы

| Проблема | Решение |
|----------|---------|
| Скрапер возвращает пустые данные | Проверь интернет, попробуй сменить user_agent, проверь логи |
| «database is locked» | Уменьши max_threads в config.yaml, проверь параллельные процессы |
| Конвейер упал | Просто перезапусти — чекпоинты продолжат с прерванного места |
| Мало компаний в городе | Проверь категории в category_cache.yaml, включи дополнительные источники |
