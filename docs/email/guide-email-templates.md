# Гайд по шаблонам писем Granite CRM

> Как писать холодные письма, follow-up и post-reply для владельцев гранитных мастерских — и почему универсальные формулы email-маркетинга здесь не работают.

---

## 1. Контекст проекта

### Что такое Granite CRM

Granite CRM — система холодного outreach для поставщика услуг ретуши портретов под гравировку на памятниках. Продукт: AI-ретушь + ручная доработка фото для мастерских по изготовлению памятников из гранита. Один исполнитель (Александр, самозанятый, НПД).

Пайплайн: парсинг (DuckDuckGo, jsprav) → дедупликация → обогащение → скоринг → экспорт (CSV/Markdown). База: 11 854 raw-компаний, 8 587 в рабочей таблице, 1 093 города.

### Как работает рассылка

Отправка писем идёт через `EmailSender` (SMTP Gmail, порт 465). Каждый email — `MIMEMultipart("alternative")` с двумя частями: plain-text (обязательно) и HTML (автогенерация из plain через `<pre>`, либо кастомный HTML-шаблон). Каждое письмо содержит tracking pixel (1x1 PNG) для фиксации открытий. Рендеринг шаблонов — через `TemplateRegistry`, который загружает `data/email_templates.json` и подставляет плейсхолдеры через `str.replace()` (литеральная подстановка, не `str.format()` — безопасно от инъекций).

Поток отправки:
1. Создание кампании → выбор шаблона + фильтры получателей + A/B темы
2. Валидация получателей (`validate_recipients`): фильтрация агрегаторов, невалидных email, SESSION_GAP (2 часа между письмами одной компании)
3. Отправка с rate limiting: задержка 45–120 сек между письмами, лимит 50/день, макс. 20/запуск
4. Логирование: `crm_email_logs` (tracking_id, ab_variant, rendered_body), `crm_touches` (исходящее касание)
5. При открытии: `maybe_create_followup_task` — создаёт задачу на follow-up через 7 дней

### Формат писем: `body_type: plain`

**Это ключевой архитектурный выбор, а не ограничение.** Все текущие шаблоны — `body_type: plain`. При отправке plain-шаблона:
- `body_text` = отрендеренный шаблон (то, что видит получатель в plain-text клиенте)
- `body_html` = автогенерация через `<pre style='font-family:sans-serif;white-space:pre-wrap'>` + tracking pixel

Это означает:
1. **Письмо выглядит как личная переписка**, а не как HTML-рассылка. Нет кнопок, нет колонок, нет логотипов — только текст. Для холодного B2B-outreach это **преимущество**, потому что получатель не видит «шаблон» и не срабатывает баннерная слепота.
2. **Маркированные списки с жирным текстом** (`**Жирный текст**`) — в plain-text отображаются как `**Жирный текст**`, то есть как мусор. Если нужен список — используйте тире или просто перечисление через запятую.
3. **HTML-теги** в plain-шаблоне отобразятся как сырой код. HTML-формат (`body_type: "html"`) поддерживается системой, но пока не используется ни в одном шаблоне.

Если в будущем появится HTML-шаблон, `EmailSender` обработает его корректно: `html_to_plain_text()` сгенерирует текстовую версию, а HTML-версия будет отправлена с tracking pixel перед `</body>`.

## 2. Портрет аудитории

### Кто получает письма

Владелец или менеджер гранитной мастерской в России. Не IT-специалист, не маркетолог, не стартапер. Человек, который:

- Каждый день работает с камнем, станками и заказчиками — не с email
- Получает 5–15 холодных писем в неделю (SEO, реклама, «увеличим продажи»)
- Сканирует входящие за 10–15 секунд — если не зацепило, удаляет
- Ценит конкретику и терпеть не может «воду» и общие фразы
- Скептически относится к слову «нейросети» в коммерческом предложении
- Работает в ритуальной сфере — где деликатность и сдержанность в общении имеют особое значение

### Психология получателя

**Главный страх**: «Мне продают то, что мне не нужно, и я потрачу время впустую». Этот страх — не паранойя, а рациональный опыт: 90% холодных писем, которые получает мастерская — мусор. Поэтому первый фильтр: «Это пишет человек или бот?». Если письмо выглядит как шаблон из CRM — удаление, даже не дочитывая.

**Главная потребность**: «Найди мне человека, который сделает ретушь нормально, быстро и без геморроя». Не «идеально», не «произведение искусства» — нормально. Качественно, чтобы заказчик не жаловался. Быстро, чтобы не тормозить процесс. Без предоплаты, без рисков, без долгих переговоров.

**Отношение к «нейросетям»**: Слово «нейросети» в предложении услуг — двояко. С одной стороны, владелец мастерской мог видеть Midjourney-картинки и понимает, что AI может что-то делать. С другой — «нейросети» ассоциируются с дешёвой автоматизацией, которая «делает плохо, но быстро». Фраза «Нейросети + ручная доработка» в текущих шаблонах — хороший баланс: показывает, что AI используется, но финальный результат — человеческая работа.

**Тон коммуникации**: Владельцы мастерских общаются между собой просто и прямо. «Здрасьте», «Сделаем», «Присылайте», «От 700 рублей». Никакого «Оптимизируйте ретушь» или «Улучшите качество портретов». Чем проще язык — тем больше доверия. Формальный тон (как в cold_email_bitrix) уместен для мастерских на Bitrix — там привыкли к деловой переписке. Но даже там — деловой, а не канцелярский.

### Сегментация получателей

Система скорит мастерских по CRM-скор (0–200) и распределяет по сегментам:
- **A (≥40)**: Мастерские с сайтом, мессенджерами, несколькими телефонами, возможно сетевые. Лучшая аудитория для холодных писем — у них есть процесс, они понимают ценность аутсорса ретуши.
- **B (25–39)**: Средние мастерские. Есть сайт или мессенджер. Могут быть заинтересованы, но нужно убедить.
- **C (15–24)**: Мелкие мастерские без сайта, с одним телефоном. Низкий приоритет для рассылки.
- **D (<15)**: Скорее всего агрегаторы или нецелевые. Обычно фильтруются на этапе `validate_recipients`.

Воронка follow-up построена на `funnel_stage`: `new` → `email_sent` → `email_opened` → `tg_sent` → `wa_sent` → (финальное письмо или ответ). Каждый переход имеет рекомендованную задержку и канал:

| Стадия | Дней до след. касания | Канал | Шаблон |
|---|---|---|---|
| new | 0 | email | cold_email_v1 |
| email_sent | 4 | tg | tg_intro |
| email_opened | 2 | tg | tg_intro |
| tg_sent | 4 | wa | wa_intro |
| wa_sent | 7 | email | follow_up_email_v1 |

## 3. Принципы писем для этой аудитории

### Принцип 1: Человек пишет человеку

Холодное письмо должно выглядеть так, будто Александр сел и написал его вручную конкретному человеку. Не как будто CRM-система сгенерировала текст по шаблону и разослала 500 адресатам. Это значит:

- Без маркированных списков с форматированием
- Без «Представьте: …» и других эмоциональных крючков
- Без «Мы» и «наши услуги» — один человек, который делает работу
- Без двоеточий в заголовках, которые разбивают тему на «рекламный крючок + оффер»
- С естественными переходами между мыслями, а не блоками «Услуги / Гарантия / CTA»

### Принцип 2: Конкретика > обещания

| Плохо | Хорошо |
|---|---|
| «Идеальные гравировки» | «Ретушь под ваш станок (лазер/ударник)» |
| «Решаю любые сложности» | «Старые фото 80-х, фото на документах, групповые» |
| «Молниеносная ретушь» | «12–24 часа, срочно — 3–6 часов» |
| «Высокое качество» | «Нейросети + ручная доработка» |
| «Оптимизируйте ретушь» | «Аутсорс ретуши — вы занимаетесь производством» |

Конкретика создаёт доверие. Обещания — сомнение. Получатель думает: «Если он обещает "идеальные гравировки", он, скорее всего, преувеличивает. А если говорит "12–24 часа, от 700₽" — он знает, о чём говорит».

### Принцип 3: Мягкий CTA, а не pressure-selling

| Плохо | Хорошо |
|---|---|
| «Не теряйте время!» | «Присылайте фото, если интересно» |
| «Не упустите возможность!» | «Готов сделать 1–2 пробных бесплатно» |
| «Хотите убедиться?» | «Начнём с бесплатной пробы» |

Владелец мастерской — не потребитель, которого можно «закрыть на сделку». Он бизнесмен, который принимает решение рационально. Лучший CTA для холодного письма — предложить безрисковый первый шаг: «Присылайте фото, покажу результат. Бесплатно, без обязательств». Это не давление — это приглашение.

### Принцип 4: Длина имеет значение

Холодное письмо — не лендинг. У получателя 5–10 секунд внимания. Оптимальная длина:

| Тип шаблона | Целевая длина | Обоснование |
|---|---|---|
| Cold email | 60–80 слов | Первое касание — только суть |
| Follow-up | 40–60 слов | Напоминание — короче первого письма |
| Post-reply (цена/сроки) | 30–50 слов | Ответ на конкретный вопрос — сразу к делу |
| Post-reply (interested) | 50–70 слов | Нужно объяснить процесс, но без воды |
| TG/WA intro | 25–35 слов | Мессенджер — ещё короче |
| TG/WA follow-up | 15–25 слов | Минимально |

### Принцип 5: Тон зависит от контекста

Три уровня формальности для разных шаблонов:

**Неформальный** (cold_email_v1, v2, marquiz, мессенджеры): «Здравствуйте!», «Беру сложные случаи», «Присылайте фото». Разговорный, но уважительный. Как если бы вы писали знакомому мастеру.

**Деловой** (cold_email_bitrix, reply_price_question): «Предлагаю сотрудничество», «Что делаю:», «Работаю официально». Структурированный, но без канцелярита. Как деловое предложение от поставщика.

**Сдержанный** (follow-up, reply_timing): «Писал на прошлой неделе», «Обычно отдаю заказ в течение 12–24 часов». Минимум эмоций, максимум информации. Как напоминание от партнёра.

## 4. Разбор текущих шаблонов

### 4.1 cold_email_v1 — Основной оффер

**Тема**: «Подготовка фото под гравировку»
**Длина**: ~75 слов

**Что работает**:
- Тема предметная, не-рекламная — выглядит как деловое письмо
- «Ищу контакты мастерских» — честная формулировка, не «Предлагаю»
- «Беру сложные случаи» — конкретика, а не обещания
- «1–2 пробных бесплатно» — безрисковый CTA
- «Если не актуально — ответьте "нет", больше не напишу» — уважение к получателю

**Что улучшить**:
- Нет «оплата после результата» — это один из сильнейших аргументов, и он есть в cold_email_marquiz, но не в v1
- «Здравствуйте.» с точкой — суховато, лучше «Здравствуйте!» или «Добрый день!»
- Можно добавить «под ваш станок» — конкретика, которую ценят мастера

### 4.2 cold_email_marquiz — Сложные случаи

**Тема**: «Ретушь портретов для вашей мастерской»
**Длина**: ~80 слов

**Что работает**:
- Перечисление конкретных сложных случаев (фото 80-х, на документах, групповые) — попадает в боль
- «Оплата после результата для новых клиентов» — мощный аргумент доверия
- «Начнём с бесплатной пробы» — мягкий CTA

**Что улучшить**:
- Тема чуть длиннее и рекламнее, чем v1 — «для вашей мастерской» выглядит как массовая рассылка
- Фраза «Беру всё что сложно» — без запятой после «всё», выглядит как опечатка
- Нет упоминания станка/технологии — для мастерских это важно

### 4.3 cold_email_bitrix — Для мастерских на Bitrix

**Тема**: «Аутсорс ретуши под гравировку»
**Длина**: ~85 слов

**Что работает**:
- Структурированный список «Что делаю:» — уместен для бизнес-аудитории
- «Ретушь под конкретный станок и технологию (лазер / ударный)» — супер-конкретика
- «Замена фона, одежды, монтаж, сборка в полный рост» — полный спектр
- «Для партнёрских мастерских — индивидуальные условия» — долгосрочная перспектива

**Что улучшить**:
- «Здравствуйте.» с точкой — суховато
- Нет упоминания оплаты после результата (есть в marquiz, нет здесь)
- «Срок в день обращения» — менее конкретно, чем «12–24 часа, срочно — 3–6 часов»

### 4.4 cold_email_v2 — Сезонная нагрузка (планируемый)

**Тема**: «Тест ретуши (в день обращения)»
**Длина**: ~60 слов

**Что работает**:
- Попадает в конкретную боль — сезонная загрузка
- «Оплата — только после того, как вы одобрите результат» — сильный аргумент
- CTA мягкий и конкретный: «Присылайте фото, если интересно»
- Подпись через `{from_name}` — персонализация

**Что улучшить**:
- Шаблон ещё не добавлен в `email_templates.json`
- Нет цены (даже «от 700₽») — получатель не знает, о каком порядке сумм речь
- Нет упоминания города — потеряна локальная привязка

### 4.5 follow_up_email_v1 — Напоминание

**Тема**: «Re: {original_subject}»
**Длина**: ~40 слов

**Что работает**:
- «Re:» — продолжает тред, повышает шанс открытия
- Короткий — получатель не чувствует давления
- «Первый портрет бесплатно» — напоминает об оффере

**Что улучшить (критично)**:
- **«Не хочу надоедать» — слабейшая фраза в шаблоне.** Она сразу ставит отправителя в позицию извиняющегося. Если вы не хотите надоедать — зачем пишете? Получатель считывает неуверенность. Эту фразу нужно убрать без замены.
- Слишком мало контекста — получатель мог забыть, о чём было первое письмо. Нужно одно предложение с сутью.
- «Просто оставлю ссылку на примеры» — пассивная формулировка. Лучше активная: «Вот ссылка на примеры».

### 4.6 tg_intro / wa_intro

**Длина**: ~30 слов

**Что работает**:
- Достаточно коротко для мессенджера
- «AI-ретушь» — объясняет метод
- «Результат за 10 минут» — конкретика
- «Первый портрет бесплатно» — CTA
- Закрытый вопрос «Интересно?» / «Пришлю примеры?» — приглашает к диалогу

**Что улучшить**:
- «Принимаю любое качество фото» — можно усилить: «Старые фото, фото на документы — любое»
- Для WA: «Здравствуйте!» более формально, чем «Добрый день!» в TG — это ок, WA чуть более деловой

### 4.7 tg_follow_up / wa_follow_up

**Длина**: ~15 / ~10 слов

**Что работает**:
- Очень коротко — минимум нагрузки на получателя
- Прямой вопрос «Актуально ли?» — чёткий CTA

**Что улучшить**:
- Пустой вопрос без ценностной привязки — получатель может подумать «Актуально ли что? Я забыл». Добавить одно слово-напоминание: «про ретушь фото для гравировки»
- wa_follow_up слишком короткий — «Добрый день, писал вам ранее про ретушь. Актуально?» — лучше, чем «Актуально?» без контекста

### 4.8 Post-reply шаблоны (reply_interested, reply_price_question, reply_timing, reply_send_examples)

Эти шаблоны ещё не добавлены в `email_templates.json`, но описаны в `docs/EMAIL_TEMPLATES.md`. Они используются через API `/companies/{id}/reply` и `/companies/{id}/reply/preview`.

**Общий принцип**: post-reply — это ответ на конкретный вопрос. Получатель уже проявил интерес. Не нужно продавать заново — нужно ответить на вопрос и продвинуть к следующему шагу.

**reply_price_question** — лучший из текущих шаблонов по структуре: цена в первом же предложении, гарантия оплаты после результата, CTA «присылайте фото». Не нужно добавлять преамбулу «Благодарю за интерес» — это +3 секунды до нужной информации.

**reply_interested** — хороший процесс (1→2→3), но «Мой процесс» звучит как «Моя методика». Лучше «Как работаем» или просто описать шаги без заголовка.

**reply_timing** — идеальный: короткий, конкретный, с CTA. Не трогать.

**reply_send_examples** — отличный ход: «Лучше один раз увидеть на СВОЁМ фото». Это работает лучше, чем просто ссылка на портфолио.

## 5. Архитектура воронки и шаблоны

### Воронка в Granite CRM

```
new ──[email, cold_email_v1]──→ email_sent ──[4 дня, tg, tg_intro]──→ tg_sent
                                                                       │
email_sent ──[открыл письмо]──→ email_opened ──[2 дня, tg, tg_intro]──┘
                                                                       │
tg_sent ──[4 дня, wa, wa_intro]──→ wa_sent ──[7 дней, email, follow_up]──→ стоп
```

Каждый переход логируется в `crm_touches` и обновляет `funnel_stage` в `crm_contacts`. Follow-up очередь доступна через `GET /api/v1/followup` с фильтрами по городу и сегменту.

### Как шаблоны взаимодействуют с воронкой

1. **Cold email** — первое касание. Цель: не продать, а вызвать интерес. Получатель должен подумать: «Нормально пишет, не спамит. Может, и правда нормально делает. Попробую — всё равно бесплатно».

2. **TG/WA intro** — второе касание через мессенджер. Цель: персонализировать контакт. Если мастерская увидит сообщение в Telegram, это ближе к «человек пишет человеку», чем email. Но мессенджер — более инвазивный канал, поэтому текст должен быть ещё короче и мягче.

3. **Follow-up email** — финальное касание. Цель: напомнить о себе тем, кто открыл письмо, но не ответил. Не «дожать», а «напомнить». Если после follow-up нет ответа — компания получает стадию `wa_sent` и остаётся в покое. Больше писем не будет (автоматически).

4. **Post-reply** — ответ на входящее. Цель: быстро и точно ответить на вопрос, продвинуть к тестовому портрету. Не «продать», а «помочь принять решение».

### Важные ограничения воронки

- **SESSION_GAP**: 2 часа между письмами одной компании. Защита от случайной двойной отправки.
- **stop_automation**: флаг в `crm_contacts`. Если установлен — автоматизация блокируется. Устанавливается при отписке или жалобе на спам.
- **unsubscribe_token**: каждый контакт имеет уникальный токен для отписки. Ссылка `{unsubscribe_url}` генерируется автоматически и подставляется в каждый шаблон. Без contact (нет записи в `crm_contacts`) письмо не отправляется — требование 152-ФЗ.
- **Daily limit**: 50 писем/день (разогрев), 200 (рабочий). Защита от попадания в спам-лист.
- **Rate limiting**: 45–120 сек между письмами. Случайная задержка в диапазоне имитирует поведение человека.

## 6. A/B тестирование

### Как работает A/B в Granite CRM

При создании кампании задаются `subject_a` и `subject_b`. Распределение — детерминированное: `MD5(company_id) % 2`. Один и тот же company_id ВСЕГДА получает один и тот же вариант, что исключает перекосы при перезапуске кампании.

Тело письма (`body`) одно и то же для обоих вариантов — тестируется только тема. Это правильно: при одновременном тестировании темы и тела невозможно понять, что именно повлияло.

Статистика A/B доступна через `GET /campaigns/{id}/ab-stats` — отправки, открытия, ответы по каждому варианту.

### Текущие A/B волны (из EMAIL_TEMPLATES.md)

| Волна | Вариант A | Вариант B |
|---|---|---|
| 1 | «Подготовка фото под гравировку» | «Ретушь портретов для вашей мастерской» |
| 2 | «Аутсорс ретуши под гравировку» | «Бесплатный тест: покажу результат за 2 часа» |
| 3 | «Тест ретуши (в день обращения)» | «Мастерские в {city_locative} экономят время на ретуши» |

### Принципы A/B тем для этой аудитории

**Тестируйте один параметр за раз**:
- Предметность vs персонализация (Волна 1)
- Деловой тон vs выгодный оффер (Волна 2)
- Скорость vs локальная привязка (Волна 3)

**Не тестируйте одновременно**:
- Длину темы и содержание (длинная тема может проиграть не из-за содержания, а из-за обрезания в почтовом клиенте)
- Эмоциональные крючки и предметность (это разные подходы к копирайтингу, и смешивать их в A/B бессмысленно)

**Минимальная выборка**: при open rate 15–20% и количестве отправок 20/запуск нужно 5–10 запусков кампании (= 100–200 получателей), чтобы получить статистически значимый результат. При меньшей выборке A/B данные информативны, но не окончательны.

**Что НЕ тестировать**: агрессивные маркетинговые заголовки типа «Ваши гравировки станут идеальными». Это не гипотеза, а риск попадания в спам. A/B тестирование спамных заголовков покажет высокий open rate на короткой дистанции и убьёт репутацию домена на длинной.

## 7. Разбор рекомендаций Manus

### Суть рекомендаций Manus

Manus предлагает применить к каждому шаблону формулу: эмоциональный крючок → маркированный список → агрессивный CTA. Конкретно:

1. Заменить заголовки на маркетинговые («Ваши гравировки станут идеальными», «Решаю любые сложности»)
2. Добавить «Представьте: …» в начало каждого письма
3. Переструктурировать текст в маркированные списки с **жирным текстом**
4. Заменить мягкие CTA на агрессивные («Не теряйте время!», «Не упустите возможность!»)
5. Увеличить длину всех писем на 40–125%
6. Заменить «я» на «мы/наши»

### Почему эти рекомендации не работают для Granite CRM

**Причина 1: Формат.** Все шаблоны — `body_type: plain`. Маркированные списки с жирным текстом (`**текст**`) отобразятся как мусор. Если переключить шаблоны на HTML, письмо сразу выглядит как рассылка — а это убивает главный актив: сходство с личной перепиской.

**Причина 2: Ниша.** Ритуальная сфера — не та область, где уместны «Представьте: каждая гравировка — произведение искусства». Владелец мастерской работает с горем людей. Высокий пафос вызывает отторжение, а не интерес.

**Причина 3: Аудитория.** Владелец мастерской — не потребитель. Ему не нужны «идеальные гравировки» — ему нужен надёжный исполнитель, который не подведёт. Агрессивные CTA вызывают защитную реакцию: «Меня продавливают — значит, что-то не так с продуктом».

**Причина 4: Длина.** Follow-up вырос на 125%. Человек, который не ответил на первое письмо, не будет читать длинное напоминание. Post-reply стали длиннее на 40–70%, при этом ответ на вопрос «Сколько стоит?» начинается с «Благодарю за интерес к нашим услугам» вместо цены.

**Причина 5: «Мы» вместо «я».** Александр — самозанятый, один человек. «Наши услуги», «наше портфолио» от одного человека создаёт ощущение подделки. Получатель видит несоответствие между «мы» и единственным контактом в подписи. «Я» — честно и вызывает доверие.

### Что стоит взять из аудита

| Рекомендация | Вердикт | Почему |
|---|---|---|
| Убрать «Не хочу надоедать» из follow-up | ✅ Взять | Слабая позиция, извиняющийся тон |
| Добавить «оплата после результата» в cold_email_v1 | ✅ Взять | Сильнейший аргумент доверия, которого нет в v1 |
| Усилить CTA в follow-up (мягко) | ✅ Взять | Напомнить суть + CTA, но без давления |
| Город в теме письма | ⚠️ Частично | Работает для некоторых тем, но не для всех |
| Добавить «Моя гарантия: оплата после результата» | ⚠️ Осторожно | Слово «гарантия» — ок, но без пафоса |
| Маркированные списки с жирным | ❌ Отклонить | Не работает в plain, выглядит как рассылка |
| Эмоциональные крючки | ❌ Отклонить | Неуместны в ритуальной нише |
| Агрессивные CTA | ❌ Отклонить | Вызывает отторжение у B2B-аудитории |
| Удлинение текстов | ❌ Отклонить | Короткое = лучше для cold email |
| «Мы/наши» вместо «я» | ❌ Отклонить | Нечестно для самозанятого |

## 10. Справочник плейсхолдеров

| Плейсхолдер | Описание | Пример | Подстановка |
|---|---|---|---|
| `{from_name}` | Имя отправителя | Александр | `os.environ["FROM_NAME"]` |
| `{city}` | Город (именительный) | Екатеринбург | `company.city` |
| `{city_locative}` | Город (предложный, после «в») | Екатеринбурге | `get_locative(city)` из `city_declensions.json` |
| `{company_name}` | Название компании | Гранит-М | `company.name_best` |
| `{website}` | Сайт компании | granit-m.ru | `company.website` |
| `{original_subject}` | Тема исходного письма | Вопрос по ретуши | Из `CrmTouchRow` (последнее входящее) |
| `{unsubscribe_url}` | Ссылка на отписку | https://… | `base_url + /api/v1/unsubscribe/{token}` |
| `{contact_name}` | Имя контакта | (пока не используется) | Зарезервировано |
| `{phone}` | Телефон компании | +7 (343) … | `company.phones[0]` |

**Важно**: `{city_locative}` вычисляется через `get_locative()` с fallback на именительный падеж, если города нет в словаре (1 093 города). После предлога «в» ВСЕГДА используйте `{city_locative}`.

## 11. Если в будущем нужен HTML-шаблон

Система поддерживает `body_type: "html"`. При отправке HTML-шаблона:
- `body_html` = отрендеренный шаблон с tracking pixel
- `body_text` = автогенерация через `html_to_plain_text()` из `granite/utils.py`
- Значения плейсхолдеров экранируются через `html.escape()` для предотвращения XSS

Если решите добавить HTML-шаблон:
1. Используйте inline-стили (не `<style>` блоки — почтовые клиенты их вырезают)
2. Таблицы для лейаута (не flexbox/grid — Outlook не поддерживает)
3. Максимальная ширина: 600px
4. Шрифт: sans-serif (Arial, Helvetica)
5. Кнопка CTA: `<table>` с фоновым цветом, не `<a>` с CSS
6. Всегда тестировать в Gmail, Outlook, Яндекс.Почта, Mail.ru

Но помните: **HTML-шаблон = рассылка**. Получатель видит дизайн и сразу понимает «Это массовая рассылка, мне отправили шаблон». Для холодного outreach plain-text — осознанный выбор, а не ограничение. HTML имеет смысл только для post-reply (когда контакт уже тёплый) или для newsletter-подписок (когда человек сам подписался).
