## 1. Обзор системы

RetouchGrav CRM — инструмент для аутрича гранитных мастерских. Она автоматически собирает данные о мастерских из открытых источников, оценивает их по скорингу и помогает управлять воронкой продаж — от первого контакта до заключения сделки.

**Ключевые возможности:**
- Автоматический сбор и обогащение данных о мастерских (телефоны, email, Telegram, CMS)
- Скоринг и сегментация A/B/C/D для приоритизации контактов
- Воронка продаж с 9 стадиями и автоматическим follow-up
- Email-кампании с отслеживанием открытий
- Batch-операции (массовая пометка спама, подтверждение, слияние дубликатов)

**Бизнес-контекст:** Зачем нужна эта система и для кого — в [project-context.md](../project-context.md).

## 2. Навигация

Боковая панель (слева) содержит основные разделы:

| Раздел | Что делать |
|--------|-----------|
| **Компании** | Основной рабочий экран — поиск, фильтрация, batch-операции |
| **На проверке** | Компании, требующие ручной проверки |
| **Follow-up** | Кому писать сегодня |
| **Задачи** | Список задач с привязкой к компаниям |
| **Шаблоны** | Управление email-шаблонами (plain и HTML) |
| **Кампании** | Создание и запуск email-рассылок |
| **Пайплайн** | Мониторинг и запуск скрапинга по городам |
| **Статистика** | Воронка, сегменты, метрики |

Внизу панели: переключатель темы (светлая/тёмная), кнопка «Админ», индикатор статуса бэкенда.

**Индикатор статуса:**
- 🟢 Online — бэкенд работает, БД подключена
- 🟡 Degraded — бэкенд доступен, но БД не отвечает
- 🔴 Offline — бэкенд недоступен

Проверяется автоматически каждые 30 секунд.

## 3. Типичные рабочие сценарии

### Сценарий 1: «Нужно отправить письмо 50 мастерским в Москве»

1. Запусти пайплайн: **Пайплайн** → выбери «Москва» → **Запустить**
2. Дождись завершения (10-30 минут, зависит от города)
3. Перейди в **Компании** → фильтр: Город = Москва, Сегмент = A
4. Выбери компании чекбоксами (или «выбрать все» в заголовке)
5. Перейди в **Кампании** → **Создать кампанию**
6. Заполни: Название = «Холодные лиды МСК», Шаблон = cold_email_1, Фильтры: город = Москва, сегмент = A
7. Нажми **Создать** → **Запустить**
8. Следи за прогрессом (обновляется автоматически)

### Сценарий 2: «Нужно проверить, кто открыл письмо»

1. Перейди в **Компании**
2. Фильтр: Воронка = «Письмо открыто»
3. Открытые письма также видны в карточке компании (колонка «Last contact»)

Альтернативно: **Статистика** → график «Воронка» показывает распределение по стадиям.

### Сценарий 3: «Нужно написать в Telegram компании»

1. Перейди в **Компании** → фильтр: Telegram = ✓
2. Кликни на компанию → в карточке (Sheet) нажми иконку Telegram
3. Откроется `t.me/` чат в новой вкладке
4. После отправки сообщения — нажми **Написали в TG** в карточке, чтобы обновить воронку

Примечание: Telegram может быть заблокирован РКН. Проверь доступность перед массовой отправкой.

### Сценарий 4: «Нужно отфильтровать горячие лиды»

1. Перейди в **Компании**
2. Нажми **Пресеты** → выбери «Горячие лиды»
3. Или вручную: Telegram ✓, Score ≥ 40

Также можно сохранить свой пресет: настрой фильтры → **Пресеты** → «Сохранить текущие».

### Сценарий 5: «Нужно загрузить HTML-шаблон из monument-web/email/»

1. Скачай `email-improved-light.html` из репозитория `monument-web`
2. Убедись, что в HTML нет `<div style="display: none"...>`, все стили inline, лейаут на `<table>`
3. Перейди в **Шаблоны** → нажми **Создать** или отредактируй существующий
4. Выбери файл через файловый аплоад (.html)
5. Укажи: channel = email, body_type = html
6. Сохрани

Подробнее: [email-sending.md](email-sending.md)

### Сценарий 6: «Нужно пометить спамом агрегаторов»

1. Перейди в **Компании** → фильтр: Сеть = ✓
2. Или: **На проверке** — агрегаторы часто попадают сюда автоматически
3. Войди в режим **Админ** (кнопка внизу боковой панели)
4. Выбери компании чекбоксами → **В спам** → причина: «Агрегатор»
5. Подтверди

## 4. Страница «Компании»

### Поиск

Поле поиска (правый верхний угол) — работает по названию компании, нечувствителен к регистру, поддерживает частичные совпадения.

### Фильтры

Панель фильтров под заголовком. Активные фильтры отображаются как цветные pills с кнопкой «Сбросить».

**Группа «Классификация»:** Сегмент (A/B/C/D/Spam), Воронка
**Группа «География»:** Регион, Город, CMS, Источник
**Группа «Каналы связи»:** Telegram, WhatsApp, Email, VK, Сайт, Адрес (трипозиционные: да/нет/любой)
**Группа «Свойства»:** Сети, Marquiz, На проверке, Автопауза, Спам, Удалённые, TG Trust, Score

Все фильтры комбинируются через **И**, кроме сегментов и городов (они — через **ИЛИ**).

### Batch-операции

Требуют режима администратора:

1. Выдели компании чекбоксами
2. Внизу появится плавающая панель: «Выбрано: N» + кнопки
3. **В спам** — выбрать причину → подтвердить
4. **Подтвердить** — снять флаг «На проверке»

### Карточка компании (Sheet)

Клик по строке → боковая панель с подробностями: контакты, сегмент, Score, CMS, TG Trust, заметки.

Действия: **Редактировать**, **В спам**, **Пометить дубль**

## 5. Воронка продаж

| Стадия | Описание | Как попадает |
|--------|----------|-------------|
| **Новый** | Нет касаний | Автоматически при создании |
| **Письмо отправлено** | Отправлено email | При отправке кампании |
| **Письмо открыто** | Получатель открыл | Tracking pixel |
| **Написали в TG** | Отправлено в Telegram | Вручную или через API |
| **Написали в WA** | Отправлено в WhatsApp | Вручную или через API |
| **Ответили** | Компания ответила | Вручную |
| **Заинтересованы** | Проявили интерес | Вручную |
| **Не интересно** | Отказались | Вручную |
| **Недоступен** | Не удалось связаться | Вручную |

Стадии обновляются вручную (через карточку компании) или автоматически (при отправке писем через кампанию).

## 6. Follow-up

Страница **«Follow-up»** — приоритетная очередь для повторного контакта. Система автоматически определяет, когда и через какой канал лучше написать снова.

**Карточка follow-up** содержит: название компании, время следующего контакта, рекомендованный канал, сегмент.

**Действия:**
- **Telegram** / **WhatsApp** / **Email** — открыть канал связи
- **✓ Сделано** — записать касание, убрать из очереди

Список обновляется автоматически каждые 60 секунд.

## 7. Кампании

### Создание кампании

1. **Кампании** → **Создать кампанию**
2. Название, шаблон письма, фильтры (город, сегмент)
3. **Создать** → кампания появится как черновик

### Запуск

Нажми **Запустить** — начнётся отправка. Прогресс обновляется автоматически (SSE). Между письмами — пауза 45–120 секунд (настраивается через EMAIL_DELAY_MIN/MAX). Дневной лимит — 50 писем (EMAIL_DAILY_LIMIT, глобальный по всем кампаниям). Лимит получателей за запуск — 10 (MAX_SENDS_PER_RUN). При достижении дневного лимита кампания переходит в статус paused_daily_limit. График прогрева — см. dev plan v13, секция 5.2.

### Статусы

| Статус | Описание |
|--------|----------|
| Черновик | Можно удалить или запустить |
| Запущена | Идёт отправка |
| Пауза | Приостановлена, можно возобновить |
| paused_daily_limit | Дневной лимит писем достигнут, кампания на паузе |
| error | Ошибка при отправке |
| Завершена | Все письма отправлены |

## 8. Пайплайн

Страница **«Пайплайн»** — запуск и мониторинг скрапинга по городам.

1. Выбери регион и город
2. Нажми **Запустить**
3. Следи за прогрессом в таблице (обновляется каждые 10 секунд)

**Стадии:** Начало → Сбор → Дедуп → Обогащение → Готов

Альтернативно — через CLI: `uv run cli.py run "Город"`

## 11. FAQ

### Фильтры не показывают результаты

Проверь pills в заголовке панели — нет ли забытых фильтров. Нажми «Сбросить» для очистки.

### Batch-панель не появляется

Кликни именно на чекбокс (клик по строке открывает Sheet). Нужно быть в режиме администратора.

### Кнопки «В спам» / «Подтвердить» отсутствуют

Нужен режим администратора. Нажми «Админ» в боковой панели и введи пароль.

### Админ-токен истёк

Токен действует 30 минут. Обратный отсчёт на кнопке «Админ».

### Нет email-шаблонов при создании кампании

Шаблоны нужно создать заранее: **Шаблоны** → **Создать** или через API `POST /templates`.

### Backend показывает «Offline»

Проверь, запущен ли бэкенд: `uv run cli.py api --port 8000`. Статус проверяется каждые 30 секунд через `/health`.

### Как посмотреть логи

```bash
tail -n 50 data/logs/granite.log
```
