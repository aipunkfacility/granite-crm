# Granite CRM — шпаргалка по командам и БД

Все команды запускаются из папки `granite-crm/`.

---

## База: как она устроена

- **Файл БД:** `data/granite.db` (SQLite)
- **Режим:** WAL — позволяет читать и писать одновременно, не блокируя друг друга
- **Миграции:** Alembic (папка `alembic/`)
- При первом запуске любой команды таблицы создаются автоматически
- Конфиг берётся из `config.yaml` (путь к БД, настройки скрапинга, пресеты и т.д.)

## Команды CLI

### 1. Подготовка (первый запуск)

```bash
# Накатить справочник городов в БД (берётся из data/regions.yaml)
uv run cli.py seed-cities
```

После этого можно смотреть статусы.

### 2. Посмотреть что в БД

```bash
# Статус по городам: сколько сырых, сколько компаний, сколько обогащённых
uv run cli.py cities-status

# Городов, которые не нашлись в справочнике (нужно разбираться руками)
uv run cli.py unmatched
```

Пример вывода `cities-status`:
```
Астраханская область:
  + * Астрахань: raw=45 comp=38 enriched=35
    * Знаменск: raw=0 comp=0 enriched=0
```
- `raw` — сырые данные со скрапинга
- `comp` — после дедупликации
- `enriched` — после обогащения (Telegram, сайт, скоринг)
- `+` — есть обогащённые данные
- `*` — город был переназначен (дубль имени в другом регионе)

### 3. Запуск пайплайна (скрапинг + дедуп + обогащение)

```bash
# Один город
uv run cli.py run Астрахань

# Все города сразу (1098 штук — долго)
uv run cli.py run all

# Перезапустить город с нуля (удалить старые данные)
uv run cli.py run Астрахань --force

# Пропустить скрапинг, взять из кэша, только обогащение
uv run cli.py run Астрахань --no-scrape

# Перепровести только обогащение (скрапинг и дедуп не трогать)
uv run cli.py run Астрахань --re-enrich
```

Флаг `--force` — удаляет всё по городу и начинает сначала. Используй если данные кривые или нужно обновить.

### 4. Экспорт данных

```bash
# Экспорт в CSV
uv run cli.py export Астрахань
uv run cli.py export all

# Экспорт в Markdown
uv run cli.py export Астрахань --format md

# Экспорт по пресету (список пресетов в config.yaml → export_presets)
uv run cli.py export-preset Астрахань hot_leads
uv run cli.py export-preset all with_telegram
```

**Доступные пресеты:**
| Пресет | Что выгружает |
|--------|--------------|
| `hot_leads` | Есть TG/WA + высокий CRM-скор |
| `high_score` | Сегмент A |
| `with_telegram` | Все у кого нашли Telegram |
| `cold_email` | Нет мессенджеров, но есть Email |
| `manual_search` | Нет мессенджеров — нужен прозвон |
| `full_dump` | Всё что обогащено |

### 5. Миграции (управление структурой БД)

```bash
# Применить все миграции (довести до актуальной версии)
uv run cli.py db upgrade

# Посмотреть текущую версию
uv run cli.py db current

# Посмотреть историю миграций
uv run cli.py db history

# Откатить на одну миграцию назад
uv run cli.py db downgrade -1

# Создать новую миграцию (если меняли модели в database.py)
uv run cli.py db migrate "added new column"

# Проверить, нужны ли миграции (сравнивает модели с реальной БД)
uv run cli.py db check

# Пометить БД как актуальную без наката миграций
uv run cli.py db stamp head
```

**Когда это нужно:** обычно — никогда. Миграции автоматически применяются при старте. Ручное управление нужно только если ты менял код моделей (`database.py`) и хочешь накатить изменения на существующую БД.

### 6. API сервер

```bash
# Запустить CRM (FastAPI) на порту 8000
uv run cli.py api

# Другой порт
uv run cli.py api --port 3000

# С hot-reload (для разработки)
uv run cli.py api --reload
```

## Таблицы в БД (кратко)

| Таблица | Зачем |
|---------|-------|
| `raw_companies` | Сырые данные со скрапинга (до дедупликации) |
| `companies` | Основная таблица компаний (после дедупа) |
| `enriched_companies` | Обогащённые данные: мессенджеры, CMS, скоринг |
| `cities_ref` | Справочник городов |
| `unmatched_cities` | Города, которых нет в справочнике |
| `crm_contacts` | CRM-карточки: воронка, метрики касаний |
| `crm_touches` | Лог каждого касания (письмо, звонок, сообщение) |
| `crm_templates` | Шаблоны сообщений |
| `crm_email_logs` | Отправленные письма + tracking |
| `crm_tasks` | Задачи (follow-up, звонки) |
| `crm_email_campaigns` | Email-кампании |

## Типичный сценарий работы

**С нуля (чистая БД):**
```bash
uv run cli.py seed-cities          # 1. Справочник городов
uv run cli.py run Астрахань        # 2. Собрать и обогатить
uv run cli.py cities-status        # 3. Проверить результат
uv run cli.py export Астрахань     # 4. Выгрузить в CSV
```

**Добавить новый город:**
```bash
uv run cli.py run Волгоград
uv run cli.py cities-status
```

**Обновить данные по городу:**
```bash
uv run cli.py run Астрахань --force
```

**Бэкап БД перед экспериментами:**
```bash
backup_db.bat  (или просто двойной клик)
```

**Посмотреть городам которых не разобрали:**
```bash
uv run cli.py unmatched
```

## Полезные настройки в config.yaml

- `database.path` — путь к файлу БД (по умолчанию `data/granite.db`)
- `scraping.max_threads` — сколько потоков скрапинга (по умолчанию 3)
- `scraping.request_delay` — задержка между запросами (по умолчанию 1.5 сек)
- `enrichment.max_concurrent` — потоков обогащения (по умолчанию 3)
- `export_presets.*.filters` — SQL-фильтры для экспорта (можно добавлять свои)
