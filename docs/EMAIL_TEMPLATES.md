# Email-шаблоны: Granite CRM

> Каталог всех email-шаблонов для рассылок и post-reply ответов.
> **Источник истины:** этот файл. Тексты правит Александр прямо здесь.
> Рабочие шаблоны загружаются из `data/email_templates.json` (TemplateRegistry).
> Для перезагрузки без рестарта: `POST /api/v1/templates/reload` или `uv run cli.py templates-reload`
>
> Плейсхолдеры: `{city}`, `{city_locative}`, `{company_name}`, `{from_name}`, `{unsubscribe_url}`, `{original_subject}`
>
> `{city_locative}` — город в предложном падеже (Москве, Казани), подставляется автоматически
> из словаря `data/city_declensions.json` через `granite/city_declensions.py`.
> После предлога «в» **всегда** используйте `{city_locative}`, а не `{city}`.

## 5. Волновые темы для A/B тестирования

При создании кампании можно задать `subject_a` и `subject_b` для A/B теста. Тело письма одно — `template_name`.

### Волна 1: Основной оффер

| Вариант | Тема | Гипотеза |
|---------|------|----------|
| A | «Подготовка фото под гравировку» | Нейтральная, профессиональная |
| B | «Ретушь портретов для вашей мастерской» | Акцент на услугу |

### Волна 2: Сложные случаи

| Вариант | Тема | Гипотеза |
|---------|------|----------|
| A | «Аутсорс ретуши под гравировку» | Конкретная деловая услуга |
| B | «Бесплатный тест: покажу результат за 2 часа» | Акцент на бесплатный тест |

### Волна 3: Сезонная нагрузка *(планируемый шаблон)*

| Вариант | Тема | Гипотеза |
|---------|------|----------|
| A | «Тест ретуши (в день обращения)» | Акцент на скорость |
| B | «Мастерские в {city_locative} экономят время на ретуши» | Локальная привязка + выгода |

## 6. Порядок работы с шаблонами

```bash
# 1. Отредактировать шаблоны прямо в этом файле (docs/EMAIL_TEMPLATES.md)
vim docs/EMAIL_TEMPLATES.md

# 2. Обновить email_templates.json
vim data/email_templates.json

# 3. Перезагрузить без рестарта сервера
uv run cli.py templates-reload
# или через API: POST /api/v1/templates/reload

# 4. Проверить текущие шаблоны
curl http://localhost:8000/api/v1/templates
```

## 7. Справочник плейсхолдеров

| Плейсхолдер | Описание | Пример подстановки |
|---|---|---|
| `{from_name}` | Имя отправителя | Александр |
| `{city}` | Название города (именительный падеж) | Екатеринбург |
| `{city_locative}` | Город в предложном падеже (после «в») | Екатеринбурге |
| `{company_name}` | Название компании | Гранит-М |
| `{original_subject}` | Тема исходного письма (для Re:) | Вопрос по ретуши |
| `{unsubscribe_url}` | Ссылка на отписку | https://… |

**Важно:** `{city_locative}` вычисляется автоматически через `get_locative(city)` из модуля `granite/city_declensions.py` с данными из `data/city_declensions.json` (1093 города). После предлога «в» всегда используйте `{city_locative}`, а не `{city}`. Если город отсутствует в словаре — fallback на именительный падеж.
