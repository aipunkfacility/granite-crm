## 1. Обзор системы

Granite CRM — специализированная система управления базой потенциальных клиентов для рынка ритуальных услуг (гранитные мастерские, производство памятников). Система автоматически собирает данные из открытых источников, дедуплицирует записи, обогащает информацию (Telegram, CMS, TG Trust) и предоставляет инструменты для работы с воронкой продаж.

### Источники данных

| Источник | Описание |
|----------|----------|
| `jsprav` | Жёлтые страницы России (jsonld + playwright fallback) |
| `web_search` | Веб-поиск (Google SERP) |
| `2gis` | 2GIS Catalog API / Crawlee |
| `yell` | Yell.ru (Crawlee + Playwright) |
| `avito` | Avito объявления |
| `google_maps` | Google Maps (grid search) |
| `jsprav_playwright` | Fallback-скрапер Жёлтых страниц через Playwright |

## 2. Навигация и боковая панель

Боковая панель (слева) содержит основные разделы системы:

| Раздел | Описание |
|--------|----------|
| **Компании** | Основной список компаний с фильтрами, сортировкой и batch-операциями |
| **На проверке** | Очередь компаний, требующих ручной проверки |
| **Follow-up** | Приоритетная очередь для повторного контакта |
| **Задачи** | Список задач с возможностью отметки выполнения |
| **Кампании** | Управление email-рассылками |
| **Пайплайн** | Мониторинг и запуск скрапинг-пайплайна |
| **Статистика** | Аналитика: воронка, сегменты, метрики |

Внизу боковой панели расположены:
- **Переключатель темы** (светлая / тёмная)
- **Кнопка «Админ»** — вход в режим администратора (см. раздел 7)
- **Индикатор статуса backend** — реальная проверка доступности сервера:
  - 🟢 **Online** — сервер работает, БД подключена
  - 🟡 **Degraded** — сервер доступен, но БД не отвечает
  - 🔴 **Offline** — сервер недоступен
  - ⏳ **Проверка...** — идёт проверка соединения

Статус проверяется автоматически каждые 30 секунд.

## 3. Страница «Компании»

Основной рабочий экран CRM. Содержит полный список компаний с возможностями фильтрации, сортировки и массовых операций.

### Поиск

Поле поиска находится в правом верхнем углу заголовка страницы. Поиск работает по названию компании (нечувствителен к регистру, поддерживает частичные совпадения). Введите часть названия — список обновится автоматически.

### Панель фильтров

Панель фильтров расположена под заголовком и состоит из четырёх групп. Заголовок панели показывает активные фильтры в виде цветных меток (pills), общее количество найденных компаний и кнопку «Сбросить».

#### Группа «Классификация»
- **Сегмент** — кнопки A / B / C / D / Spam (можно выбрать несколько, работают как ИЛИ)
- **Воронка** — выпадающий список стадий воронки

#### Группа «География и платформа»
- **Регион** — выпадающий список регионов
- **Город** — множественный выбор (выбранные города отображаются как pills с крестиком)
- **CMS** — выпадающий список CMS-систем (Bitrix, WordPress и т.д.)
- **Источник** — выпадающий список источников данных (jsprav, web_search и т.д.)

#### Группа «Каналы связи»
Трипозиционные переключатели (да / нет / любой):
- **Telegram** — есть/нет Telegram-аккаунт
- **WhatsApp** — есть/нет WhatsApp
- **Email** — есть/нет email
- **VK** — есть/нет группа ВКонтакте
- **С сайтом** — есть/нет сайт
- **С адресом** — есть/нет физический адрес

Значения переключателей:
| Состояние | Что означает |
|-----------|-------------|
| Не выбрано (серое) | Фильтр не применяется — показываем все |
| Зелёный ✓ | Только компании, у которых этот канал есть |
| Красный ✗ | Только компании, у которых этого канала нет |

#### Группа «Свойства»
- **Сети** — является ли компания сетью филиалов
- **Марquiz** — установлен ли виджет Marquiz на сайте
- **На проверке** — требует ли ручной проверки
- **Автопауза** — приостановлена ли автоматизация
- **Спам** — показать/скрыть/только спам
- **Удалённые** — показать удалённые компании
- **TG Trust** — минимальный уровень доверия Telegram (0–3)
- **Score** — диапазон CRM-оценки (от — до)

### Пресеты

Кнопка **«Пресеты»** (рядом с поиском) позволяет сохранять и быстро применять наборы фильтров.

**Системные пресеты (★):** Горячие лиды, Высокий скор, С Telegram, Холодный Email, Ручной поиск, Полный дамп.

**Пользовательские пресеты (🔖):** Настройте фильтры и нажмите «Сохранить текущие». Хранятся в браузере.

### Таблица компаний

| Колонка | Описание | Сортировка |
|---------|----------|------------|
| ☐ | Чекбокс для массового выделения | — |
| Название | Имя компании | Да |
| Город | Город компании | Да |
| Сегмент | A / B / C / D / Spam | Да |
| Score | Числовая CRM-оценка | Да |
| Воронка | Стадия воронки | Да |
| Контакт | Иконки Telegram, телефон, сайт, VK | — |
| Последний контакт | Время последнего контакта | Да |
| Сеть | Является ли сетью | Да |
| Обновлено | Дата последнего обновления | Да |

#### TG Trust — индикаторы
| Уровень | Цвет | Значение |
|---------|------|----------|
| 0 | 🔴 | Мёртвый аккаунт (бот или удалён) |
| 1 | 🟠 | Частичный (аватарка есть, но неактивен) |
| 2 | 🔵 | Живой (активен) |
| 3 | 🟢 | Активный (постоянная активность) |

### Сортировка

Кликните на заголовок сортируемой колонки: первый клик — по убыванию, повторный — по возрастанию.

### Управление колонками

Кнопка **«Колонки»** (справа над таблицей) — скройте/покажите нужные колонки. Настройка сохраняется в браузере.

### Карточка компании (Sheet)

Кликните на строку компании, чтобы открыть боковую панель с подробной информацией: название, сегмент, Score, адрес, каналы связи (Telegram + TG Trust, WhatsApp, VK, Email, телефоны), сайт, CMS, Marquiz, дата обогащения.

Действия в Sheet:
- **Редактировать** — изменить данные компании
- **В спам** — пометить как спам (нужна причина)
- **Пометить дубль** — указать дубликат другой компании

## 6. Фильтры — подробный справочник

### Быстрые сценарии

**Горячие лиды для Telegram-рассылки:** Пресет «Горячие лиды» или вручную: Telegram ✓, Score ≥ 40.

**Холодные email:** Telegram ✗, WhatsApp ✗, Email ✓, сегмент B/C.

**Проверить качество данных по городу:** Выберите город, сравните «С адресом» ✓ и ✗.

**Сравнить источники:** Источник = jsprav, затем web_search — сравните Score.

Все фильтры комбинируются через **И**, кроме сегментов и городов (они работают через **ИЛИ**).

## 7. Администрирование

### Вход в режим администратора

1. Нажмите **«Админ»** в боковой панели
2. Введите пароль (`GRANITE_ADMIN_PASSWORD` из `.env`)
3. Кнопка станет зелёной с обратным отсчётом (30 минут)
4. После истечения — войдите заново

### Batch-операции

1. **Выделите компании** чекбоксами в таблице (или «выбрать все» в заголовке)
2. Внизу появится плавающая панель: «Выбрано: N» + кнопки
3. **В спам** — выбрать причину (агрегатор, закрылась, не та категория, дубликат, другое) → подтвердить
4. **Подтвердить** — снять флаг «На проверке»
5. Результаты: ✅ Готово или ⚠️ Частично (N из M обработано)
6. Список обновляется автоматически, выделение снимается

## 8. Очередь проверки

Страница **«На проверке»** показывает компании, помеченные системой для ручной проверки. Причины: конфликты адресов при дедупликации, обрезка контактов при слиянии, иные автоматические флаги.

### Действия с компанией

| Действие | Описание |
|----------|----------|
| **Подтвердить** | Снять флаг проверки, компания возвращается в общий список |
| **В спам** | Пометить как спам (нужна причина), компания скрывается |
| **Дубликат** | Пометить как дубль другой компании (указать ID оригинала) |

В режиме администратора доступны кнопки массовых действий: «Подтвердить все» и «В спам все».

## 9. Follow-up

Страница **«Follow-up»** — приоритетная очередь компаний, которым нужен повторный контакт. Система автоматически определяет, когда и через какой канал лучше связаться повторно.

### Карточка follow-up

Каждая карточка содержит:
- Название компании (ссылка на карточку)
- Время следующего контакта
- Рекомендуемое действие
- Сегмент (A/B/C/D)

### Действия

| Кнопка | Что делает |
|--------|------------|
| **Telegram** | Открывает `t.me/` чат с компанией |
| **WhatsApp** | Открывает `wa.me/` чат |
| **Email** | Открывает `mailto:` для отправки письма |
| **✓ Сделано** | Записывает касание (touch) в историю, убирает из очереди |

Список обновляется автоматически каждые 60 секунд.

## 11. Кампании

Страница **«Кампании»** — управление массовыми email-рассылками по сегментам клиентов.

### Создание кампании

1. Нажмите **«Создать кампанию»**
2. Заполните форму:
   - **Название** — например, «Холодные лиды МСК»
   - **Шаблон письма** — выберите из списка email-шаблонов (создаются через API)
   - **Город** (фильтр, необязательно) — ограничить получателей по городу
   - **Сегмент** (фильтр, необязательно) — A/B/C/D
3. Нажмите **«Создать»** — кампания появится как черновик

### Управление кампанией

Каждая кампания-карточка показывает:

| Элемент | Описание |
|---------|----------|
| Статус | Черновик / Запущена / Пауза / Завершена |
| Прогресс | Прогресс-бар отправки (отправлено / всего) |
| Охват | Количество отправленных писем |
| Open Rate | Процент открытых писем |
| Ответов | Количество ответов |

**Кнопки действий:**
- **Запустить** — начать рассылку (для черновиков и приостановленных)
- **Пауза** — приостановить рассылку (для запущенных)
- **🗑 Удалить** — удалить черновик

### Статусы кампании

| Статус | Описание |
|--------|----------|
| Черновик | Кампания создана, но не запущена. Можно удалить. |
| Запущена | Идёт отправка писем. Прогресс обновляется автоматически (каждые 5 сек). |
| Пауза | Рассылка приостановлена. Можно возобновить. |
| Завершена | Все письма отправлены. |

### Шаблоны писем

Шаблоны создаются через API (POST `/templates`). Каждый шаблон содержит: название, канал (email), тему и тело письма. В выпадающем списке создания кампании показываются только email-шаблоны.

## 12. Пайплайн

Страница **«Пайплайн»** — мониторинг и управление процессом сбора и обработки данных по городам.

### Запуск скрапинга

В верхней части страницы — блок «Запуск скрапинга»:

1. **Регион** — выберите регион из выпадающего списка (фильтрует города)
2. **Город** — начните вводить название, появится прокручиваемый список с поиском. Рядом с каждым городом указан его регион.
3. **Запустить** — запуск пайплайна для выбранного города

Если пайплайн уже запущен для этого города — кнопка будет неактивна.

### Сводные карточки

| Карточка | Описание |
|----------|----------|
| Всего городов | Количество городов в базе |
| В обработке | Количество городов с активным пайплайном |
| База компаний | Общее количество компаний |

### Таблица статусов

| Колонка | Описание |
|---------|----------|
| Город | Название города (🔄 если в обработке) |
| Стадия | Сбор / Дедуп / Обогащение / Готов / Начало |
| Прогресс | Прогресс-бар обогащения (%) |
| Raw | Количество сырых записей |
| Компании | Количество компаний после дедупликации |
| Обогащено | Количество обогащённых компаний |

### Фильтр по региону

Выберите регион в блоке «Запуск скрапинга» — таблица и карточки отфильтруются. Для сброса — выберите «Все регионы».

### Стадии пайплайна

| Стадия | Описание |
|--------|----------|
| Начало | Город есть в справочнике, но данных ещё нет |
| Сбор | Собраны сырые записи (raw_companies) |
| Дедуп | Записи дедуплицированы, созданы компании |
| Обогащение | Компании обогащаются (TG, CMS, скоринг) |
| Готов | Обогащение завершено |

Данные обновляются автоматически каждые 10 секунд.

## 15. Решение частых проблем

### Фильтры не показывают результаты

Проверьте pills в заголовке панели — нет ли забытых фильтров. Нажмите «Сбросить» для очистки всех фильтров.

### Batch-панель не появляется

Убедитесь, что вы кликнули именно на чекбокс в начале строки (клик по строке открывает Sheet). Если чекбоксы выделены, но панель не видна — обновите страницу.

### Кнопки «В спам» / «Подтвердить» отсутствуют

Batch-операции требуют режима администратора. Нажмите «Админ» в боковой панели и введите пароль.

### Админ-токен истёк

Токен действует 30 минут. Обратный отсчёт на кнопке «Админ». Когда время истечёт — войдите заново.

### Нет email-шаблонов при создании кампании

Шаблоны создаются через API: `POST /api/v1/templates`. Без шаблонов создать кампанию нельзя.

### Источник показывает 0 компаний

Если данные загружены до добавления фильтра по источнику, система использует fallback-запрос к таблице raw_companies. Если и там пусто — данных из этого источника ещё не загружалось.

### Backend показывает «Offline»

Проверьте, запущен ли backend-сервер. Статус проверяется каждые 30 секунд через `/health` endpoint. Если проблема сохраняется — проверьте переменную `NEXT_PUBLIC_API_URL`.
